Overview

CodeComplete is a powerful code generation tool designed to automatically create Data Abstraction Layer (DAL) classes and various code artifacts from SQL Server database schemas. The tool can generate code in multiple languages and formats, making it an invaluable asset for rapid application development.

Key Features:

  • Automatic generation of full CRUD (Create, Read, Update, Delete) operations
  • Support for multiple output formats: C#, TypeScript, JavaScript, Python, Java
  • Web service generation (Web Methods, WebAPI, REST)
  • UI generation (WinForms, XAML, HTML/CSS)
  • AI-assisted code generation via LM Studio integration
  • Utility tools for string manipulation, SQL formatting, and Base64 encoding

Tab 1: Connection Parameters

This is the primary configuration screen where you establish database connectivity and configure code generation options.

Database Connection Section

Connection Methods:

  • Manual Connection String - Paste a complete connection string and press Enter
  • Server Enumeration - Click the binoculars button to discover available SQL Servers on the network
  • Manual Server Name - Type in a specific server name

Dropdown Controls:

  • Servers Dropdown - Select from enumerated SQL Server instances
  • Databases Dropdown - Choose a database from the selected server
  • Tables Dropdown - Pick a table or view for code generation

Authentication

  • User Textbox - SQL Server login ID
  • Password Textbox - SQL Server password (masked)
  • SSPI Button - Use Windows Authentication instead of SQL credentials

Code Generation Options Panel

A comprehensive set of checkboxes that control what code gets generated:

Generate Add Method - Creates INSERT functionality
Generate FastAdd Method - Creates optimized INSERT with fewer parameters
Generate Delete Method - Creates DELETE functionality
Generate Read Method - Creates SELECT by ID functionality
Generate ReadAsDataSet Method - Creates method returning DataSet
Generate Update Method - Creates UPDATE functionality
Generate Web Methods - Creates SOAP web service methods
Generate Deactivate Method - Creates soft delete functionality
INotifyPropertyChanged - Implements property change notification
Post Process WS Result List - Additional processing for web service results
XAML Form or UserControl - Controls XAML output type
Generate JavaScript UI - Creates JavaScript-based user interface
Generate Insert Statements - Creates SQL INSERT scripts

Additional Configuration

  • Interface Object Prefix - Prefix for interface/lookup field names
  • CMD Timeout - SQL command timeout value in seconds
  • Show Connection String Button - Displays the derived connection string
  • Derived Connection String Display - Shows the constructed connection string

Data Grid

Displays the table schema including column names, data types, and properties for the selected table.

Tab 2: Base Code C#

Generates a complete C# Data Abstraction Layer class for the selected database table.

Generated Code Includes:

  • Class definition with properties for each table column
  • Multiple constructors (default, parameterized, from DataRow, from SqlDataReader)
  • CRUD methods:
    • Read(id) - Retrieve single record by ID
    • ReadAsDataSet(criteria) - Query returning DataSet
    • Add() - Insert new record
    • FastAdd() - Optimized insert
    • Update() - Update existing record
    • Delete(id) - Remove record
  • Property validation and type conversion
  • Database connection management

Controls:

  • Class Name Textbox - Override the default generated class name
  • Regenerate Base Class Button - Regenerate code with current settings
  • Line Numbers Checkbox - Toggle line number display in editor
  • Code Folding Checkbox - Enable/disable code folding regions
  • Save to File Button - Export generated C# code to .cs file
Note: The generated code uses ADO.NET with SqlClient for database operations and includes proper error handling and connection disposal.

Tab 4: Web Method

Generates SOAP web service methods (ASMX-style) for exposing database operations via web services.

Generated Web Methods:

  • [WebMethod] Read(id) - Returns serialized object
  • [WebMethod] ReadList(criteria) - Returns list of objects
  • [WebMethod] Add(object) - Inserts record via web service
  • [WebMethod] Update(object) - Updates record via web service
  • [WebMethod] Delete(id) - Deletes record via web service

Controls:

  • Line Numbers Checkbox
  • Code Folding Checkbox
  • Save Web Methods Button

These methods are ready to be inserted into an ASP.NET Web Service (.asmx) file.

Tab 5: WebAPI

Features:

Controls:

  • Line Numbers Checkbox

Tab 17: AI

AI-assisted code generation using LM Studio integration for advanced code generation in multiple languages.

Supported AI Code Generation:

  • Python - Generate Python data classes and database access code
  • Java - Generate Java entity classes and DAO patterns
  • RESTful API - AI-generated REST endpoint implementations

Controls:

  • LM Studio Address Textbox - URL of your LM Studio instance (e.g., http://localhost:1234)
  • Connect to LM Studio Button - Establish connection to AI service
  • Make Base Python Button - Generate Python data access classes
  • Make Base Java Button - Generate Java entity and DAO classes
  • Make Restful API Button - Generate AI-powered REST endpoints
  • Working Label - Shows status during AI generation
  • Output Editor - Displays AI-generated code
  • Line Numbers Checkbox
  • Code Folding Checkbox

How AI Generation Works:

  1. Connect to LM Studio running locally or on network
  2. Select a table from the Connection Parameters tab
  3. Choose the desired output language/format
  4. Click the generation button
  5. The AI analyzes the table schema and generates appropriate code
Requirements: LM Studio must be running and accessible. The AI uses the table schema from your currently selected database table to generate contextually appropriate code.

AI Generation Advantages:

  • More intelligent naming conventions
  • Context-aware code patterns
  • Modern framework integration (Flask, Spring Boot, etc.)
  • Best practices implementation
  • Comments and documentation generation

General Workflow

  1. Connect to Database - Use Tab 1 to establish connection
  2. Select Table - Choose the database table for code generation
  3. Configure Options - Set checkboxes for desired methods and features
  4. Generate Code - Navigate to desired tab to view generated code
  5. Customize - Edit generated code in the syntax-highlighted editor if needed
  6. Save - Use the Save button to export code to files
  7. Integrate - Add generated files to your project

Common Editor Features

Most code display tabs use the Scintilla editor component with these features:

  • Syntax Highlighting - Language-specific color coding
  • Line Numbers - Toggle via checkbox
  • Code Folding - Collapse/expand code regions
  • Search/Replace - Standard text editing capabilities
  • Copy/Paste - Easy code extraction

Tips and Best Practices

  • Test Connection First - Always verify database connectivity before attempting code generation
  • Review Generated Code - While the tool generates functional code, always review for your specific requirements
  • Save Connection Strings - Store frequently used connection strings for quick access
  • Use Class Name Override - Customize class names to match your project's naming conventions
  • Leverage Multiple Tabs - Generate both frontend and backend code simultaneously
  • AI for Complex Scenarios - Use AI generation for more sophisticated patterns
  • Stringify for Embedding - Use the Stringify tab when embedding large text blocks in code
  • SQL Beautifier - Clean up generated or logged SQL before using in stored procedures
